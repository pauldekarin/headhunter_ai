# Stage 2 — Расширение

**Цель**: внешний AI-канал через [[MCP]] для Claude Desktop / Claude Code, автоматическая отправка с лимитами, дедупликация, шаблоны промптов, e2e-тесты, авто-обновления.

## Задачи

### 2.1 MCP server — `L`

MCP-сервер как параллельный канал работы с вакансиями для внешних AI-клиентов (Claude Desktop, Claude Code). Экспонирует tools поверх существующего backend HTTP API. См. [[MCP]].

MCP — отдельный сервис в `services/mcp/`, общается с backend только через HTTP API. Tools: `list_pending_vacancies`, `get_vacancy(id)`, `get_user_context()`, `submit_cover_letter(vacancy_id, text)`, `get_application_status(vacancy_id)`. Транспорт stdio + streamable HTTP.

Дополняет [[AI Layer]] из [[Stage 1 - MVP#1.12 AI Layer — multi-provider + fallback — L|1.12]]: пользователь либо приносит ключ к API-провайдеру (Stage 1 путь), либо подключает Claude Desktop по подписке (Stage 2 путь).

**AC**: Claude Desktop с конфигом видит tools; вызов tool пишет в БД через HTTP API; собственный UI продолжает работать через REST/WS параллельно.

**Зависимости**: [[Stage 1 - MVP]] 1.7, 1.10, 1.11, 1.12.

**Стек**:

| Задача | Библиотека | Почему | Альтернативы |
|---|---|---|---|
| MCP SDK | **`mcp`** (Anthropic official, Python) | Официальный, все транспорты | — |
| Декларативные tools | **`fastmcp`** | Удобная обёртка с декораторами `@tool` | ручной handler-стиль через `mcp` |

### 2.2 Промпт-шаблоны и версионирование — `M`

Промпты в `~/.headhunter_ai/prompts/*.yaml` ([[Storage]]), версионирование, A/B-тест двух версий.

**AC**: Можно завести 2 промпта, метрика «принято пользователем» сохраняется.

**Зависимости**: 2.1.

### 2.3 Auto-submit режим с лимитами — `M`

Token-bucket: N откликов/час, M/день. UI показывает оставшийся бюджет. См. [[Anti-bot]].

**AC**: Превышение лимита блокирует отправку с понятной ошибкой.

**Зависимости**: [[Stage 1 - MVP]] 1.8.

**Стек**:

| Задача | Библиотека | Почему |
|---|---|---|
| Token bucket | **`limits`** (Python) | Готовая реализация |

### 2.4 Дедупликация — `S`

Не парсить и не откликаться на вакансии, по которым уже есть `Application`. См. [[Domain Model]].

**AC**: Повторный поиск не возвращает уже обработанные.

**Зависимости**: [[Stage 1 - MVP]] 1.5.

---

### 2.5 Captcha-handling — `M`

Детект по селектору/URL → пауза очереди → нотификация в [[UI]] → ручное решение в открытом Chromium → resume. См. [[Anti-bot]].

**AC**: Ручная капча решается за < 30 сек, очередь продолжается.

**Зависимости**: [[Stage 1 - MVP]] 1.1, 1.6.

---

### 2.6 Audit log + экспорт — `S`

Полный лог всех действий: парсинг, генерация, отправка, ошибки. Экспорт CSV/JSON.

**AC**: Экспорт за период работает; лог содержит trace-id.

**Зависимости**: [[Stage 1 - MVP]] 1.5.

Playwright (TS) гоняет полный сценарий по mock hh.ru (или dev-стенду). См. [[Testing]].

**AC**: CI запускает 3 e2e-сценария.

**Зависимости**: [[Stage 1 - MVP]] 1.10.

**Стек**:

| Задача | Библиотека | Почему |
