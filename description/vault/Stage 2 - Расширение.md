# Stage 2 — Расширение

**Цель**: внутренняя AI-абстракция ([[AI Layer]]), автоматическая отправка с лимитами, дедупликация, шаблоны промптов, e2e-тесты, авто-обновления.

## Задачи

### 2.1 LLM-абстракция через LiteLLM — `M`

Слой `AILayer.generate(prompt, ctx) -> str` с поддержкой Anthropic, OpenAI, Ollama. Конфиг в settings. См. [[AI Layer]].

**AC**: Переключение провайдера в [[UI]] без рестарта; fallback при ошибке.

**Зависимости**: [[Stage 1 - MVP]] 1.6.

**Стек**:

| Задача | Библиотека | Почему | Альтернативы |
|---|---|---|---|
| LLM proxy | **LiteLLM** | 100+ провайдеров, OpenAI-совместимый API, retry/fallback/caching | langchain (тяжёлый); прямые SDK (дублирование) |
| Локальный LLM | **Ollama** | Простой headless хостинг | llama.cpp |

---

### 2.2 Промпт-шаблоны и версионирование — `M`

Промпты в `~/.headhunter_ai/prompts/*.yaml` ([[Storage]]), версионирование, A/B-тест двух версий.

**AC**: Можно завести 2 промпта, метрика «принято пользователем» сохраняется.

**Зависимости**: 2.1.

---

### 2.3 Auto-submit режим с лимитами — `M`

Token-bucket: N откликов/час, M/день. UI показывает оставшийся бюджет. См. [[Anti-bot]].

**AC**: Превышение лимита блокирует отправку с понятной ошибкой.

**Зависимости**: [[Stage 1 - MVP]] 1.8.

**Стек**:

| Задача | Библиотека | Почему |
|---|---|---|
| Token bucket | **`limits`** (Python) | Готовая реализация |

---

### 2.4 Дедупликация — `S`

Не парсить и не откликаться на вакансии, по которым уже есть `Application`. См. [[Domain Model]].

**AC**: Повторный поиск не возвращает уже обработанные.

**Зависимости**: [[Stage 1 - MVP]] 1.5.

---

### 2.5 Captcha-handling — `M`

Детект по селектору/URL → пауза очереди → нотификация в [[UI]] → ручное решение в открытом Chromium → resume. См. [[Anti-bot]].

**AC**: Ручная капча решается за < 30 сек, очередь продолжается.

**Зависимости**: [[Stage 1 - MVP]] 1.1, 1.6.

---

### 2.6 Audit log + экспорт — `S`

Полный лог всех действий: парсинг, генерация, отправка, ошибки. Экспорт CSV/JSON.

**AC**: Экспорт за период работает; лог содержит trace-id.

**Зависимости**: [[Stage 1 - MVP]] 1.5.

---

### 2.7 E2E-тесты — `L`

Playwright (TS) гоняет полный сценарий по mock hh.ru (или dev-стенду). См. [[Testing]].

**AC**: CI запускает 3 e2e-сценария.

**Зависимости**: [[Stage 1 - MVP]] 1.10.

**Стек**:

| Задача | Библиотека | Почему |
|---|---|---|
| E2E | **Playwright Test** (TS) | Тот же тулинг что для парсинга |
| Mock hh.ru | **mitmproxy + HAR** или локальный fixture-сервер | Воспроизводимо |

---

### 2.8 Auto-update — `M`

Tauri updater + подпись релизов ed25519 + GitHub Releases как фид.

**AC**: Запущенное приложение получает уведомление об апдейте.

**Зависимости**: [[Stage 0 - Setup]] 0.2.

**Стек**:

| Задача | Библиотека | Почему |
|---|---|---|
| Auto-update | **`tauri-plugin-updater`** | Официальный, проверка подписи |
| Code signing | **`tauri-action`** + GitHub OIDC | Автоматизирует подпись |

---

### 2.9 Selector resilience — `M`

Snapshot-тест селекторов hh.ru, daily smoke-test в CI. См. [[Parser service]].

**AC**: При поломке селектора падает CI с понятным diff.

**Зависимости**: [[Stage 1 - MVP]] 1.3.
