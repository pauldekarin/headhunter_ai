## Технические риски

### 1. Эволюция вёрстки hh.ru
Селекторы будут ломаться.

**Митигация**: централизованный модуль селекторов в [[Parser service]] + daily smoke-test ([[Stage 2 - Расширение]] 2.9).

### 2. Бан аккаунта
Даже с stealth и лимитами hh.ru может детектить автоматизацию.

**Митигация**: дисклеймер ([[Anti-bot]]), рекомендация использовать на не-основном аккаунте, лимиты по умолчанию консервативные.

### 3. MCP-протокол молод и быстро меняется
Спецификация в активной разработке.

**Митигация**: использовать официальный SDK `mcp`, фиксировать версии, выделить адаптер для абстракции.

### 4. Chromium-профиль пользователя
Если использовать существующий профиль — можно повредить пользовательские куки/расширения.

**Митигация**: создавать **отдельный** профиль в `~/.headhunter_ai/chrome-profile/`, в который пользователь логинится один раз. См. [[Chromium]].

### 5. Sandbox Tauri
Запуск отдельных Python-процессов как sidecar требует подписи и whitelist на macOS/Windows.

**Митигация**: использовать `tauri-plugin-shell` с зарегистрированным sidecar binary; настроить code signing в CI.

### 6. Размер инсталлятора
Playwright тянет за собой ~150MB Chromium.

**Митигация**: открытый вопрос (см. [[Open Questions]] #1) — bundled vs lazy download.
