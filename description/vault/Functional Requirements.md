# Functional Requirements

## Явные (из ТЗ [[Description]])

1. Запуск [[Chromium]] и проверка авторизации на hh.ru; редирект на форму логина если не залогинен.
2. Конфигурация фильтра поиска и контекста чата (характер + знания о соискателе).
3. Парсинг вакансий со всеми полями из [[Domain Model]].
4. Стрим вакансий в реальном времени через WebSocket ([[REST]]).
5. Передача вакансии: [[Parser service]] → БД ([[Storage]]); генерация письма через [[AI Layer]] (`POST /api/v1/ai/cover_letter`) или альтернативно через внешний AI-клиент по [[MCP]] (Stage 2); запись результата в `cover_letters`.
6. Отправка отклика через [[Writer service]].
7. Отображение в [[UI]]:
    - вакансия ждёт письма
    - отклик отправлен (с возможностью посмотреть письмо)
    - ошибка

## Неявные (вытекают из задачи)

> [!info] Не упомянуто в ТЗ, но критично для usable-продукта.

- Превью и редактирование письма перед отправкой
- Очередь и приоритезация откликов
- Дедупликация (не откликаться повторно на ту же вакансию)
- Лимиты по дневной/часовой норме откликов (см. [[Anti-bot]])
- Hot-reload фильтров без перезапуска Chromium
- Экспорт истории откликов (CSV/JSON)
- Шаблоны промптов для разных категорий вакансий
- Onboarding-модал с дисклеймером о ToS hh.ru при первом запуске
