# Stage 3 — Оптимизация

Параллелизм, наблюдаемость, i18n, темы, бэкапы, plugin API.

## Задачи

### 3.1 Параллельный парсинг — `M`

До 3 вкладок Chromium параллельно для ускорения. См. [[Parser service]].

**AC**: Парсинг 100 вакансий < 2 минут.

**Зависимости**: [[Stage 1 - MVP]] 1.3.

---

### 3.2 Метрики и dashboards — `M`

Prometheus-экспортер на `localhost:9090`; шаблон Grafana dashboard. См. [[Observability]].

**AC**: Метрики: вакансий/мин, ошибки, токены, латентность LLM.

**Зависимости**: [[Stage 2 - Расширение]] 2.6.

---

### 3.3 Tracing — `M`

OpenTelemetry traces от REST до Playwright-action.

**AC**: Span'ы видны в Jaeger/Honeycomb.

**Зависимости**: 3.2.

---

### 3.4 i18n — `S`

RU/EN в [[UI]]. Переключатель языка работает, ключи в JSON.

**Зависимости**: —.

**Стек**:

| Задача | Библиотека | Почему |
|---|---|---|
| i18n UI | **paraglide-js** | Type-safe, tree-shakeable |

---

### 3.5 Темы (light/dark) — `S`

Системная + override. Переключение без перезагрузки.

**Зависимости**: —.

**Стек**: **`mode-watcher`** (Svelte).

---

### 3.6 Резервные копии — `S`

Авто-бэкап SQLite + конфигов раз в неделю. Файл в `~/.headhunter_ai/backups/`, ротация 4 шт. См. [[Storage]].

**Зависимости**: [[Stage 1 - MVP]] 1.5.

---

### 3.7 Plugin API для нестандартных фильтров — `L`

Python entry-points для кастомных селекторов/фильтров.

**AC**: Сторонний плагин из примера загружается и работает.

**Зависимости**: 3.1.
