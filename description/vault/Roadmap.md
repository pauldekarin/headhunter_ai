# Roadmap

Дорожная карта проекта [[Description|Headhunter AI]].

## Этапы

- [[Stage 0 - Setup]] — фундамент: монорепа, CI, дисклеймер
- [[Stage 1 - MVP]] — рабочий путь end-to-end через [[MCP]] + Claude Desktop
- [[Stage 2 - Расширение]] — внутренний AI ([[AI Layer]]), авто-режим, e2e-тесты
- [[Stage 3 - Оптимизация]] — параллелизм, наблюдаемость, i18n, плагины

## Критический путь MVP

```
Stage 0 → 1.1 → 1.6 → 1.7 → 1.10 → MVP RELEASE
              ↑          ↑
        1.4, 1.5, 1.8, 1.9, 1.11 — параллельно
```

**Минимум**: задачи 1.1, 1.6, 1.7. Остальное распараллеливается на 2–3 разработчика.

## Сжатый MVP (если нужно показать как можно скорее)

1.1 (BrowserCore) → 1.2 (auth) → 1.3 (parser) → 1.4 (FastAPI) → 1.5 (SQLite) → 1.6 (orchestrator) → 1.7 (MCP)

И **минимальный CLI вместо UI** — можно демо через Claude Desktop без своего фронта. UI ([[Stage 1 - MVP]] задачи 1.9–1.11) делается параллельно или во вторую очередь.

## Дальше

```
MVP ──→ 2.1 (LiteLLM) ──→ 2.2 (промпты) ──→ 2.3 (auto-submit)
    └─→ 2.4 (dedup) [parallel]
    └─→ 2.5 (captcha) [parallel]
    └─→ 2.6 (audit) [parallel]
    └─→ 2.9 (selectors) [parallel]
                            ──→ 2.7 (e2e) ──→ 2.8 (auto-update) ──→ v1.0

v1.0 ──→ Stage 3 (3.1-3.7 в основном параллельно)
```

## Принципы планирования

- **MVP не включает [[AI Layer]]** — Claude Desktop по [[MCP]] закрывает AI-нужду «бесплатно»
- **Auto-submit — только во 2-м этапе** — в MVP только ручное подтверждение, чтобы минимизировать риск бана
- **E2E-тесты — после MVP** — сначала ручная валидация, потом автоматизация
- **Параллелизм парсинга — последний этап** — сначала корректность, потом скорость
